VCSA 6.7的备份
vCenter Server Appliance 6.7支持基于文件的备份。此功能支持为vCenter Server Appliance和Platform Services Controller执行备份计划任务。支持以下协议作为备份目标HTTP，HTTPS，FTP，FTPS，SCP。
首先登录到需要备份的VMware Appliance管理界面。浏览器打开https://VCSA_IP:5480。使用root账户登录
在VMware设备管理界面的右下角，转到Backup选项卡。
单击 CONFIGURE 启动备份计划向导。
在备份计划向导中，使用支持的备份协议之一（HTTPS，HTTP，SCP，FTPS，FTP）输入备份位置。备份位置使用以下格式：Protocol://Server Address/Backup Folder/Subfolder，备份计划会自动在目标上以日期创建备份目录存放每次的备份文件。
制定备份计划，可以根据需要来定期执行备份任务，建议每天晚上执行一次。
为了防止目标空间耗尽，设置保留的备份数量，请根据需要来设置。建议保留最近一周备份数据。
创建完备份任务后，可以从 Backup 选项卡查看备份计划状态。 展开 Status 以查看其配置，还可以在右上角编辑，禁用和删除备份计划。
可以等待设置的时间自动触发备份，也可以立即执行一次备份
备份完成后，可以在目标上看到备份的数据。这样当VCSA故障的时候就可以通过备份数据进行恢复了。
VCSA 6.7的恢复
VCSA 6.7恢复过程跟新安装类似，需要使用vCenter安装ISO镜像。在安装的过程中可以通过备份文件导入恢复配置，来实现恢复到故障前的vCenter配置。
导航到安装文件的vcsa-ui-installer目录，选择执行对应操作系统的installer启动vCenter安装程序。
注意:安装程序支持macOS、Linux和Windows操作系统
选择 Restore 选选
指定存放备份数据的位置
从备份数据中选择需要恢复到的配置
提供新恢复VCSA所在的ESXi主机FQDN或IP地址等信息。然后下一步。
输入VCSA的虚拟机名称，及设置root帐号
设置VC的IP网络，可以沿用之前的IP地址，前提确保故障VC已经不可访问。
vCenter Server Appliance的部署（阶段1）已完成。 单击 CONTINUE 从备份中还原配置数据（阶段2）。
注意：如果此时选择“CLOSE”，则可以从新打开5480端口的VMware Appliance管理界面继续执行阶段2。
验证备份数据，并输入加密密码（如果之前备份设置了密码），然后下一步。
设置SSO帐户，即默认的本地管理员帐号Administrator@vsphere.local
等待几分钟，恢复将完成。
vCenter服务器恢复完成，登录vCenter后发现原来的配置信息都在。整个过程大约十几分钟。

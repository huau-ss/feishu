VCSA 小版本升级（打补丁）操作指南
本指南适用于Linux 版VCSA，包括6.5、6.7 和7.0 等多版本。小版本升级就是通常所
说的update，比如从U1 升级到U2，或者U2 升级到U2c 等。但不包括从6.5 升级到6.7 或
6.5 升级到7.0 这种跨大版本的Upgrade。这种大版本的升级，需要考虑兼容性、许可、API 等多方面因素，本文不适用。
假设当前版本为6.5 U1，现在6.5 最新版本为6.5 U3f，是否可以一次性升级还是需要
经过几个中间版本，这个是不确定的，有可能行，也有可能不行。但如果是从6.5U3 到6.5U3f
是可以确定一次性的，无需经过6.5 u3a,6.5u3b…，等。
小版本的升级也可以登陆VCSA 的5480 端口，从图形界面升级。但一般不建议，因为
图形界面看不到进度条，无法判断是否升级成功。
具体操作步骤如下：
一、登陆my.vmware.com 官网，下载补丁包
二、记录vCenter 虚拟机所在的ESXi 宿主机
[第2页]
三、将下载的升级包上传到ESXi 宿主机
四、vCenter 虚拟机正常关机（注意不是直接关闭电源）
[第3页]
五、对vCenter 虚拟机做快照
快照完成后，打开vCenter 虚拟机电源。
[第4页]
六、vCenter 虚拟机挂载升级包
七、SSH 连接到vCenter 虚拟机
（注）：如果SSH 未启用，请先登陆htttps://vcip:5480，开启SSH 登陆 [第5页] 八、执行升级命令
(1) software-packages stage --iso
(2) software-packages list –staged
(3) software-packages install –staged
注意：这三条命令需要在Command>提示符下输入，如果登陆后直接进入了#提示符下，需
要输入appliancesh 进入命令Command>提示符
software-packages stage --iso 相当于Linux 系统内的mount,正常执行后会弹出许可协议，
根据屏幕向导，一路回车，最后需要手工输入yes，表示同意许可，如果没有输入yes，后续 无法进行 [第6页]
software-packages list –-staged 查看升级包
software-packages install –-staged 进行升级（注：7.0 版本要求输入管理员密码、6.5 和6.7 版本不需要）
提示升级成功后，进入Shell ，输入reboot 重新引导系统。 如果升级意外失败或重启后有
其它问题可通过快照恢复到升级前状态。
九、后续清理
1、取消ISO 挂载
2、确定各项功能正常后，删除快照
十、Tips
对于VCSA 6.5 和6.7，可以在运行升级命令前，先停止服务，# service-control --stop --all
然后再执行software-packages 升级命令，这有助于解决某些情况下升级Hang 住的问题。 但这不适用于VCSA 7.0
